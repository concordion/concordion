Possible OGNL expressions

OGNL (Object Graph Notation Language) is the expressions used to reference properties, methods and variables inside of the concordion test.
Concordion supports complex expressions, but they should be used with caution and follow KISS principle as this is a testing code.
Listed below expressions are valid with concordion.

Examples

Literals:
  • 0
  • 0.0
  • string literal
Operators
  • 1 lt 2
  • 1 lte 2
  • 2 gt 1
  • 2 gte 1
  • 1 eq 1
  • 1 neq 2
  • true or true
  • true and true
Ternary
  • 2 gt 1 ? true : false
Variables:
  • #variable
  • #arrayVariable[0]
  • #mapVariable['value']
  • #complexVariable.subProperty
  • #complexVariable.subMethod()
  • #complexChainViaPropertyVariable.next.subProperty
  • #complexChainViaPropertyVariable.next.subMethod()
  • #complexChainViaMethodVariable.next().subProperty
  • #complexChainViaMethodVariable.next().subMethod()
Properties:
  • property
  • arrayProperty[0]
  • mapProperty['value']
  • complexProperty.subProperty
  • complexProperty.subMethod()
  • complexChainPropertyViaProperty.next.subProperty
  • complexChainPropertyViaProperty.next.subMethod()
  • complexChainPropertyViaMethod.next().subProperty
  • complexChainPropertyViaMethod.next().subMethod()
Methods:
  • method()
  • methodWithArguments()
  • methodReturningArray()[0]
  • methodReturningMap()['value']
  • methodReturningComplexType().subProperty
  • methodReturningComplexType().subMethod()
  • methodReturningComplexChainViaProperty().next.subProperty
  • methodReturningComplexChainViaProperty().next.subMethod()
  • methodReturningComplexChainViaMethod().next().subProperty
  • methodReturningComplexChainViaMethod().next().subMethod()
Mixed:
  • #var.createProperties('app').map['system_file'][0]
