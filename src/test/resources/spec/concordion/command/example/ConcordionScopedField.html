Variable Scope in Examples

Concordion has support for four types of fixture field scope.

  1. Example: fields in the fixture annotated with @ConcordionScopedField(scope = ConcordionScopedField.Scope.EXAMPLE) will be re-initialised between each test.
  2. Specification: fields in the fixture annotated with @ConcordionScopedField(scope = ConcordionScopedField.Scope.SPECIFICATION) will shared between tests.
  3. Global: fields in the fixture annotated with @ConcordionScopedField(scope = ConcordionScopedField.Scope.GLOBAL) will shared between all fixtures.
  4. Unannotated: fields in the fixture without any annotations will be re-initialised between each test. This is standard jUnit behaviour (in fact, jUnit creates a new fixture object for each test).
