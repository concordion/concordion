KeyMatch verification strategy


This file provides examples of behaviour of concordion:verifyRows command with concordion:matchStrategy="KeyMatch".
The command tries to find expected row by fields marked as a key (it must be unique). If key is found then the rest of the fields is asserted else row is reported missing.
Please note that commands are being evaluated on a copy of the rows multiple times. This means commands has to be idempotent and lightweight.
