Default verification strategy


This file provides examples of behaviour of concordion:verifyRows command with concordion:matchStrategy="Default".
This commands verify rows top to bottom without modifying order of actual rows.

Examples

Scenario Expected Actual Result
All match in same order 
<table concordion:verifyRows="#user : users" concordion:matchStrategy="Default">
    <tr>
        <th concordion:assertEquals="#user.firstName">First Name</th>
        <th concordion:assertEquals="#user.lastName">Last Name</th>
        <th concordion:assertEquals="#user.age">Age</th>
    </tr>
    <tr>
        <td>John</td>
        <td>Smith</td>
        <td>17</td>
    </tr>
    <tr>
        <td>Jane</td>
        <td>Dou</td>
        <td>22</td>
    </tr>
</table>
